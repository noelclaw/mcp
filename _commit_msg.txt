feat: deep_research v3 - Grok Live Search integration

When Grok is the active LLM provider, deep_research augments its Firecrawl
pipeline with xAI Live Search - pulling real-time results from X (Twitter),
news outlets, the web, and RSS feeds directly during synthesis. Citations
are returned and appended to the final report under Real-time sources.

New input params on deep_research:
- liveSearch (boolean) - opt-in/out. Default: auto-on when Grok is active.
- liveSearchSources (string[]) - pick from web, x, news, rss. Default web+x+news.
- liveSearchDays (number) - restrict to last N days. Default: no filter.

LLM layer (llm.ts):
- LLMOptions interface with optional liveSearch
- callLLM passes through to callGrok; other providers ignore it
- isGrokActive() helper for tools
- callGrok reads search_parameters; appends citations as sentinel block

Synthesis updates:
- Prompt instructs how to weave Live Search alongside numbered sources
- Citation style: [N] for static, (X post) / (news, outlet) inline for live
- Conflict flagging when real-time disagrees with static

Output:
- Header shows Live Search status + live-citation count
- New Real-time sources subsection grouped by X / News / Web

Bump 3.10.2 to 3.11.0.
